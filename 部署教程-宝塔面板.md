# 评分系统部署到宝塔面板教程

## 项目概述

| 组件 | 技术栈 |
|------|--------|
| 后端 | Fastify + Prisma + SQLite |
| 前端 | Vue 3 + Vite + Element Plus |
| 数据库 | SQLite（文件型，`server/prisma/dev.db`） |
| 运行端口 | 默认 3100 |

---

## 一、服务器环境准备

进入 **宝塔面板 → 软件商店 → 搜索「Node.js版本管理器」**，安装后：

1. 点击「Node.js版本管理器」→ **安装 Node.js v22**（或 18+）
2. 设为默认版本

---

## 二、本地构建

在本地项目根目录执行：

```bash
npm install
npm run build
```

构建完成后会生成：

- `server/dist/` — 后端编译产物
- `web/dist/` — 前端静态文件

---

## 三、上传文件到服务器

### 3.1 项目目录结构

假设服务器项目目录为 `/www/wwwroot/pfxt.wghappy.cn/`，需要上传以下文件：

```
pfxt.wghappy.cn/
├── package.json                # 根目录 package.json（含 workspaces 配置）
├── server/
│   ├── package.json
│   ├── dist/                   # 后端编译产物
│   ├── prisma/
│   │   ├── schema.prisma
│   │   └── dev.db              # ⭐ SQLite 数据库文件
│   └── uploads/                # 上传文件目录
└── web/
    ├── dist/                   # 前端静态文件
    └── public/                 # 模板和签名文件（后端导出需要）
        ├── 微格教学技能评价表.docx
        └── 签名/               # 评委签名图片
```

上传方式：宝塔面板文件管理器拖拽上传压缩包后解压，或使用 SFTP/SCP。

> ⚠️ **首次部署上传 `server/prisma/dev.db`，后续更新时绝对不要覆盖此文件，否则数据会丢失！**

### 3.2 安装服务端依赖

项目使用了 npm workspaces，**必须在根目录安装依赖**（不是在 server 目录）：

```bash
cd /www/wwwroot/pfxt.wghappy.cn
npm install

cd server
npx prisma generate
```

> ⚠️ 不要在 `server/` 目录下单独执行 `npm install`，workspaces 项目这样做会失败。

---

## 四、配置环境变量

在 `server/` 目录下创建 `.env` 文件：

```bash
nano /www/wwwroot/pfxt.wghappy.cn/server/.env
```

写入以下内容：

```env
DATABASE_URL="file:./prisma/dev.db"
JWT_SECRET="请替换为至少32位的随机字符串"
PORT=3100
```

> 生成随机密钥：`openssl rand -hex 32`

---

## 五、宝塔 Node 项目配置

### 5.1 添加 Node 项目

进入 **宝塔面板 → 网站 → Node项目 → 添加Node项目**：

| 配置项 | 值 |
|--------|----|
| 项目目录 | `/www/wwwroot/pfxt.wghappy.cn/server` |
| 启动选项 | `start`（即 `node dist/src/index.js`） |
| Node版本 | v22 |
| 包管理器 | npm |
| 运行用户 | www |
| 项目端口 | 3100 |

### 5.2 关于「依赖安装异常」提示

宝塔可能提示「项目依赖安装异常，请先到模块管理中重新安装依赖」。

**不要用宝塔的模块管理按钮安装依赖**，因为它会在 `server/` 目录下执行 `npm install`，workspaces 项目这样做会失败。

依赖已在第三步中通过根目录 `npm install` 安装好了，直接点**启动**即可。

## 六、创建网站 + Nginx 配置

### 6.1 创建网站

进入 **宝塔面板 → 网站 → 添加站点**：

- 域名：填写你的域名（如 `pfxt.wghappy.cn`）
- 根目录：`/www/wwwroot/pfxt.wghappy.cn/web/dist`
- PHP 版本：选「纯静态」

### 6.2 配置反向代理

点击网站 → **设置 → 反向代理 → 添加反向代理**：

| 字段 | 值 |
|------|----|
| 代理名称 | pfxt-api |
| 代理目录 | `/api/` |
| 目标URL | `http://127.0.0.1:3100` |
| 发送域名 | `$host` |

添加后，点击该代理的 **「配置文件」**，将内容替换为：

```nginx
#PROXY-START/

location ^~ /api/
{
    proxy_pass http://127.0.0.1:3100;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header Upgrade $http_upgrade;
    proxy_set_header Connection "upgrade";
    proxy_http_version 1.1;
    client_max_body_size 50m;
    proxy_read_timeout 300s;
}

#PROXY-END/
```

> ⚠️ **关键：`proxy_pass http://127.0.0.1:3100;` 末尾不能加 `/`！**
> 宝塔默认生成的配置末尾会带 `/`，这会导致 Nginx 去掉 `/api/` 前缀，后端收到错误路径，返回 400 Bad Request。**必须手动删掉这个 `/`。**

### 6.3 编辑网站主配置文件

点击网站 → **设置 → 配置文件**，在 `#引用反向代理规则` 之后、`location /` 之前添加：

```nginx
    # WebSocket 代理
    location /ws {
        proxy_pass http://127.0.0.1:3100;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_read_timeout 86400;
    }

    # SSE 连接（公共页面实时更新）
    location /api/public/sse {
        proxy_pass http://127.0.0.1:3100;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_http_version 1.1;
        proxy_set_header Connection "";
        proxy_buffering off;
        proxy_cache off;
        proxy_read_timeout 86400;
    }

    # Vue Router history 模式 fallback
    location / {
        try_files $uri $uri/ /index.html;
    }
```

### 6.4 配置 SSL

在宝塔面板点击网站 → **SSL → Let's Encrypt**，申请免费证书并开启强制 HTTPS。

## 七、更新部署流程（保留数据库）

### 7.1 本地构建

```bash
npm run build
```

### 7.2 服务器上替换文件

```
✅ 替换：server/dist/、server/package.json、server/prisma/schema.prisma、web/dist/
✅ 同步：web/public/（模板文件、签名图片等，后端导出评价表依赖这些文件）
❌ 不要覆盖：server/prisma/dev.db、server/.env、server/uploads/
```

### 7.3 服务器上执行

```bash
# 如有新依赖
cd /www/wwwroot/pfxt.wghappy.cn
npm install

# 如 schema 有变动
cd server
npx prisma generate
npx prisma db push
```

### 7.4 重启项目

在宝塔 Node 项目管理中点击 **重启**。

## 九、常见问题

### Q: 访问报 400 Bad Request

检查 Nginx 反向代理配置中 `proxy_pass` 末尾是否多了 `/`：

```nginx
proxy_pass http://127.0.0.1:3100;    ✅ 正确
proxy_pass http://127.0.0.1:3100/;   ❌ 错误
```

### Q: 报 `The table main.Activity does not exist`

数据库文件为空或路径不对。确认：

1. `.env` 中 `DATABASE_URL="file:./prisma/dev.db"`
2. `server/prisma/dev.db` 文件存在且非空（应有几百KB以上）
3. 文件权限正确：`chown www:www server/prisma/dev.db`

如需重建空数据库：

```bash
cd /www/wwwroot/pfxt.wghappy.cn/server
npx prisma db push
npx tsx prisma/seed.ts
```

### Q: WebSocket 连接失败

确保 Nginx 主配置文件中添加了 `/ws` 的代理（见第六步 6.3）。

### Q: 502 Bad Gateway

```bash
curl -s http://127.0.0.1:3100/api/health
```

常见原因：
- `.env` 中 `JWT_SECRET` 未配置或长度不足 32 位
- Node 项目未启动
- 端口被占用

### Q: 前端页面刷新后 404

确保 Nginx 配置了 `try_files $uri $uri/ /index.html;`

### Q: 宝塔提示依赖安装异常

项目使用 npm workspaces，必须在**根目录**（不是 server 目录）执行 `npm install`。忽略宝塔模块管理的提示，直接启动即可。
